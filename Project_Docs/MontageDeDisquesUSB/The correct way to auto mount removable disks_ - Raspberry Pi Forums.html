The correct way to auto mount removable disks?

  • Quote

Sat Sep 29, 2012 2:41 am

FWIW I've started with the current 2012-09-18-wheezy-raspbian.img plus updates.

I'm trying to find the correct way to setup automounting disks.
I'm attempting to make a headless server that will host some HFS+ volumes over AFP. I have Netatalk & HFSProgs working fine, the sharing all works. Journalling is off on the disks & everything is stable. I'm using it headless, so I don't want GUI based solutions unless I can set them up once.

How should I make the USB disks auto mount when connected?

FStab
I have added settings to /etc/fstab and the disk mounts via sudo mount -a
It is mounted after a reboot too, but it doesn't remount if I use umount the disk, remove and reconnect it.

The fstab entry looks like…

Code: Select all

UUID=d44b8516-bdb8-3225-82c6-6b0aa7bccc2a  /media/16  hfsplus  defaults,force  0  0
NB: 'force' is required for HFS+

Is there a deamon or service I need to start to have auto mount from fstab entries?

Autofs looks like it could be the answer I installed it, disabled the fstab entry, and added the following to /etc/auto.master

Code: Select all

/var/autofs/removable   /etc/auto.removable     --timeout=20
and /etc/auto.removable

Code: Select all

hfs      -fstype=hfsplus,rw,gid=100,umask=002,force       :/dev/sda2
I rebooted so the services can restart cleanly.

Autofs works to a point.
I can access files in the disk on request over an ssh session, but if I try to mount an AFP sharepoint that is inside the volume it appears empty at the client until I login & access files on the disk. The AFP sharepoint fails to work at the client too, the disk becomes locked & unwritable.

Is there any clean way to set this up? I'd like to simply plug the disk in & have the partitions mount, I don't mind if I can hard code them in a config file, (like fstab). Any idea what I'm missing?

That's an automounter that will detect and mount usb devices to /media/usb*.
Configuration is in /etc/usbmount/*.

Using autofs to automatically mount removable drive:

The following steps will mount my ntfs formatted usb drive to /mnt/shared

1) make sure USB drive is plugged in and then run following command to get UUID:

Code: Select all

ls -l /dev/disk/by-uuid/
Make a note o the UUID as you will need later:
[img]pi@raspberrypi ~ $ ls -l /dev/disk/by-uuid/
total 0
lrwxrwxrwx 1 root root 15 Mar 27 14:17 140A-14B7 -> ../../mmcblk0p1
lrwxrwxrwx 1 root root 15 Mar 27 14:17 f24a4949-f4b2-4cad-a780-a138695079ec -> ../../mmcblk0p2
lrwxrwxrwx 1 root root 10 Mar 27 16:18 F8D841F5D841B2A4 -> ../../sdb1
[/img]
2) download autofs

Code: Select all

sudo apt-get install autofs
3) if your USB drive is NTFS formatted you will need to download ntfs-3g:

Code: Select all

sudo apt-get install ntfs-3g
4) decide where you will want autofs to automatically create and mount your usb drive NOTE:I made it under /mnt but make sure you do not have other sub-folders in there as autofs will remove anything that is not in its config files, if you do have manually created folders in /mnt then create a new folder and use that instead of /mnt):

5) configure the autofs config file

Code: Select all

sudo nano /etc/auto.master
add following line:

Code: Select all

/mnt            /etc/auto.ntfs --timeout=60 --ghost
6) create a new file (use name you used in above step, i.e. /etc/auto.ntfs)

Code: Select all

sudo nano /etc/auto.ntfs
and add following line (replace my UUID with yours noted in step one)

Code: Select all

shared -fstype=ntfs-3g UUID=F8D841F5D841B2A4
NOTE: If your USB drive is not NTFS you can use fstype=vfat for fat or if you dont know you could try fstype=auto
7) restart autofs service and you are done!!! Navigate to /mnt/shared to see the contents of your removable USB storage

Code: Select all

sudo service autofs restart
